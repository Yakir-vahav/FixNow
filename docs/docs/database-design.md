# Database Design - FixNow

## מטרת מסד הנתונים
מסד הנתונים נועד לשמור את כל המידע של מערכת FixNow:
משתמשים, בעלי מקצוע, קריאות שירות ודירוגים.

---

## טבלת Users

טבלה זו שומרת את כל המשתמשים במערכת.

| שדה | סוג נתון | תיאור |
|---|---|---|
| id | INT | מזהה משתמש |
| fullName | VARCHAR(100) | שם מלא |
| email | VARCHAR(100) | אימייל |
| password | VARCHAR(255) | סיסמה |
| phone | VARCHAR(20) | מספר טלפון |
| role | VARCHAR(20) | סוג משתמש: customer / worker |
| createdAt | DATETIME | תאריך יצירה |

---

## טבלת ServiceRequests

טבלה זו שומרת את קריאות השירות שנפתחות על ידי לקוחות.

| שדה | סוג נתון | תיאור |
|---|---|---|
| id | INT | מזהה קריאה |
| customerId | INT | מזהה הלקוח שפתח את הקריאה |
| workerId | INT | מזהה בעל המקצוע שקיבל את הקריאה |
| profession | VARCHAR(50) | סוג בעל מקצוע |
| description | TEXT | תיאור התקלה |
| address | VARCHAR(255) | כתובת |
| phone | VARCHAR(20) | טלפון ליצירת קשר |
| status | VARCHAR(20) | Open / Accepted / Completed / Cancelled |
| createdAt | DATETIME | תאריך פתיחת הקריאה |

---

## טבלת Reviews

טבלה זו שומרת דירוגים של לקוחות על בעלי מקצוע.

| שדה | סוג נתון | תיאור |
|---|---|---|
| id | INT | מזהה דירוג |
| requestId | INT | מזהה קריאת השירות |
| customerId | INT | מזהה הלקוח |
| workerId | INT | מזהה בעל המקצוע |
| rating | INT | דירוג מ־1 עד 5 |
| comment | TEXT | הערה |
| createdAt | DATETIME | תאריך יצירת הדירוג |

---

## קשרים בין הטבלאות

- לקוח אחד יכול לפתוח הרבה קריאות שירות.
- בעל מקצוע אחד יכול לקבל הרבה קריאות שירות.
- כל קריאת שירות יכולה לקבל דירוג אחד.
- כל דירוג שייך ללקוח, לבעל מקצוע ולקריאת שירות.

---

## סטטוסים אפשריים לקריאת שירות

| סטטוס | משמעות |
|---|---|
| Open | קריאה פתוחה וממתינה לבעל מקצוע |
| Accepted | בעל מקצוע קיבל את הקריאה |
| Completed | העבודה הסתיימה |
| Cancelled | הקריאה בוטלה |
